Functions

Overview

Functions allow you to actually do things. When you implement a function, you're writing out what you want it to do. In order to make that actually happen, you have to call the function. Functions take stuff in, do things to it, and spit stuff out.

Description

Depending on what you want to do, you can make functions that will take something in and manipulate it, or take nothing in and do stuff anyway. When you want your function to take something in, you specify what type of data you want to pass in through parameters when you write out the implementation. Then, when you call the function, you use your specific data as arguments. This is clearer when we know how to create and use functions.

Making Functions

Here's how we make functions:


            function funcName(x: type, y: type): type {
                // implementation
            }

Here, x and y are parameters, and you must give them types just as you would a variable. The last type before the { is the return type of the function. Functions, after doing whatever you say, return things. Just as with everything else, you must specify the type of the thing you are returning.

Return Types of Functions

Functions can return pretty much whatever you want, and we already know we have to specify the type of whatever is returned. Possible types include:

  • number
  • string
  • boolean
  • void
    • void is the type you give when you don't want your function to return anything. Functions that don't return anything still do things, they just don't give you anything back at the end.

These aren't the only types functions can return. If something exists as a type, it can be a return type of a function.

Example Function


            function add(x: number, y: number): number {
                let z: number = x + y;
                return z;
            }

This function takes in two numbers, x and y, adds them, and returns the sum, z.

If this were a void function, then what? Void functions don't return anything, so we could either leave out the return statement or simply say return;.

Functions and Return

Functions can only return once. After a return statement is encountered and executed, the function ends. Functions can have multiple return statements within them, but every function still only returns once-a function will never execute multiple return statements.

What we put in the return statement must match the function's return type.

Calling Functions

So far, we've defined functions. Now we want to use them, so we must call them. Here's how:


            funcName(arguments);

All we do is write the function's name and then in the parentheses we put in our arguments, which are just our specific values for the parameters.

Calling the Example Function


            add(4, 3);

Here, we pass in argument 4 for parameter x and argument 3 for parameter y. This is fine since the number of arguments (2) matches the number of parameters (2). It is also necessary that the first argument has the same type (in this case, number) as the first parameter (number), and the second argument has the same type as the second parameter.

So what happens? The function returns 7, since z became 4 + 3, which evaluates to 7, and the function returns z. Return is NOT the same as print, so unless we print the output by saying print(add(4, 3));, nothing is printed. If we would write this print statement, 7 would print since the function executes and returns z, which is 7, to the place it was called, which was inside the print statement.

Note on Variables and Functions

Variable usage is limited to inside the variable's scope. A variable's scope is the space within the curly braces {} that enclose the variable. Outside this scope, variables are not accessible. So, when we declare variables within functions, they can only be used within that function, and we can't get to them anywhere else.

Anonymous Functions

Anonymous functions may sound a little mysterious, but don't worry! Anonymous functions are really just functions that are anonymous-they have no name! Other than that, they're not much different from normal functions. Anywhere we can refer to a normal function by its name we can also use an anonymous function. This is useful with variables that are assigned functions and when passing functions in as arguments to other functions or methods. Here's how we make one:


                function(parameter: type): type {
                    // implementation
                }

Now we'll look at how to assign an anonymous function to a variable. Even though our function will end with a closing curly brace }, we'll still need to end with a semicolon ; since the function is part of a variable assignment statement:


                let aNewVariable: type = function(parameter: type): type {
                    // implementation
                };

Now we'll get into how to use an anonymous function as an argument of another function or method. We can't forget to put in the closing curly brace of the function }. We also can't forget to close our parentheses of the function or method into which we are passing our anonymous function as an argument. We also can't forget to end with a semicolon ; again! Let's see this in action:


                [-2, -1, 0, 1].map(function(n: number): number {
                    return n * 2;
                });

Anonymous functions are closures, which means they can use variables declared outside of its block even if those variables aren't in block scope. Overall, anonymous functions aren't really that different from normal functions, but they have a few cool attributes like this that can sometimes make them fun and helpful to use.

Functions that Call Themselves

With functions, we can do so many fun things, such as RECURSION! YAY! When a function calls itself as part of its implementation, this is recursion. Any time we call a function, a new frame is added on the memory stack for its local variables. The local variables in a frame are independent of local variables in other frames. An important thing to remember when we're doing recursion is that we need a base case to tell the function to stop calling itself at some point, or else it will call itself forever. This would cause infinite frames to be added to the memory stack, resulting in a stack overflow error! We definitely don't want that, so we always make sure that on some conditions, the function will call itself, but for at least one condition that will at some point be true, it won't call itself anymore. Here's an example of recursion: 


            function eatChristmasCookies(stillHungry: boolean, numCookiesEaten: number): number {
                if (!stillHungry) {
                    return numCookiesEaten;
                    // this is our base case! Here, we're not calling our function, but just returning numCookiesEaten.
                } else if (numCookiesEaten >= 10) {
                    stillHungry = false;
                    return numCookiesEaten + eatChristmasCookies(stillHungry, numCookiesEaten);
                } else {
                    numCookiesEaten += 5;
                    return numCookiesEaten + eatChristmasCookies(stillHungry, numCookiesEaten);
                }
            }

            print("We've eaten " + eatChristmasCookies(true, 0) + " Christmas cookies! Yum!");

This will print out: "We've eaten 35 Christmas cookies! Yum!"

Here's how we got to 35:

When we first call our function, it returns 5 + eatChristmasCookies(true, 5); Since what we're returning contains a call to this function, we go back in and see what that call returns, and continue like this until we reach the base case and stop calling the function. Once we reach this base case, we substitute this value in for the previous call of the function in its return statement and follow this pattern until we've gotten through it all. When we do that, we're popping the last frame we added off our stack in memory. Here's one way to keep track of what's returned:

5 + eatChristmasCookies(true, 5)

10 + eatChristmasCookies(true, 10)

10 + eatChristmasCookies(false, 10)

10

Now that we've hit the end we can start filling our values in and going back up. We do this by replacing the function call with what it returns:

5 + eatChristmasCookies(true, 5)

10 + eatChristmasCookies(true, 10)

10 + 10

We'll keep moving up.

5 + eatChristmasCookies(true, 5)

10 + 10 + 10

We're still going back up until we hit the top.

5 + 10 + 10 + 10

This evaluates to 35! We did it!

Note: the return statements in the else if block and the final else block are the same, so we could've just said it once at the end of our function to do the same thing:


            function eatChristmasCookies(stillHungry: boolean, numCookiesEaten: number): number {
                if (!stillHungry) {
                    return numCookiesEaten;
                } else if (numCookiesEaten >= 10) {
                    stillHungry = false;
                } else {
                    numCookiesEaten += 5;
                }
                return numCookiesEaten + eatChristmasCookies(stillHungry, numCookiesEaten);
            }

Both ways of writing the function accomplish the same thing. Writing the recursive return statement separately in both the else if and the else block helps us keep track of which cases are our base cases and which are our recursive cases, which can help us make sure we don't forget to include a base case. However, writing it as one recursive return statement at the end is more concise.
